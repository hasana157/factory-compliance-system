==========================================================================
HOW TO INITIALIZE THIS PROJECT
==========================================================================

# 1. Clone or create repo
git init factory-compliance-system
cd factory-compliance-system

# 2. Create folders
mkdir -p src/detection src/severity src/escalation src/reports src/dashboard
mkdir -p data/train data/test
mkdir -p outputs exports/
mkdir -p tests docs

# 3. Initialize Python backend
touch src/__init__.py
touch src/main.py src/config.py src/database_init.py
touch requirements.txt

# 4. Initialize each module
touch src/detection/__init__.py src/detection/detector.py src/detection/models.py
touch src/severity/__init__.py src/severity/classifier.py
touch src/escalation/__init__.py src/escalation/router.py
touch src/reports/__init__.py src/reports/generator.py
# ... etc

# 5. Initialize React frontend
cd src/dashboard
npx create-react-app . --template typescript
# ... or setup manually

# 6. Add git files
touch .gitignore .env.example README.md
echo "node_modules/" >> .gitignore
echo "__pycache__/" >> .gitignore
echo ".env" >> .gitignore
echo "outputs/*" >> .gitignore

# 7. Git setup
git add .
git commit -m "Initial project structure"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/factory-compliance-system.git
git push -u origin main

==========================================================================
QUESTIONS YOU'LL ASK AS YOU BUILD
==========================================================================

Q: Should I fine-tune YOLOv8?
A: No. Pre-trained on COCO is fine. Document that you'd fine-tune with more time.

Q: Do I need authentication in dashboard?
A: No. Single-user dashboard is fine. Optional: add simple login.

Q: Should the detection be perfect?
A: No. ~70-80% accuracy is acceptable. Document limitations.

Q: Do I need to deploy?
A: No. Local running system is sufficient.

Q: What if module X is not working?
A: Document it in README. Show what you tried. Honesty = respect.

Q: Should I use cloud services?
A: No. Keep it local. Easier to run, easier to evaluate.

Q: How polished should the UI be?
A: Functional > beautiful. A working ugly dashboard beats a pretty broken one.

Q: Do I need a database?
A: Yes. SQLite is required for report persistence (Module 4).

Q: Can I use different tech stack?
A: Yes, but stick to Python backend + React frontend. Matches spec.

Q: What if I run out of time?
A: Submit what you have. Document trade-offs. Honesty matters.
